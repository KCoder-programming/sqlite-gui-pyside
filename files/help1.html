Introduction to SQLite

SQLite is a lightweight, serverless, and self-contained SQL database engine. It stores the entire database as a single file on disk, making it highly portable and easy to use for small to medium applications.

SQL Command Categories in SQLite

SQL commands are categorized into:

  • DDL — Data Definition Language
  • DML — Data Manipulation Language
  • DQL — Data Query Language
  • TCL — Transaction Control Language (limited support)

1. DDL (Data Definition Language)

CREATE TABLE

Used to create a new table in the database.

CREATE TABLE students (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    age INTEGER,
    email TEXT
);

With condition to create table only if it do not exists

CREATE TABLE IF NOT EXISTS customers_temp (
    customer_id INTEGER PRIMARY KEY,
    username TEXT
);

ALTER TABLE

  • Modify an existing table (SQLite allows only adding columns).

    ALTER TABLE students ADD COLUMN phone TEXT;

  • Rename Table

    ALTER TABLE old_table_name RENAME TO new_table_name;

  • Rename Column

    ALTER TABLE students RENAME COLUMN id TO user_id;

DROP

Delete an existing table permanently.

DROP TABLE students;

Note: SQLite does not support TRUNCATE. Use DELETE FROM tablename; instead to remove all rows.

Delete column:

ALTER TABLE students DROP COLUMN phone

2. DML (Data Manipulation Language)

INSERT INTO

Insert new records into a table.

INSERT INTO students (name, age, email) VALUES ('John Doe', 20, 'john@example.com');

Copy Data From the Old Table to the New Table

INSERT INTO customers_temp(customer_id, username) SELECT id, name FROM students;

UPDATE

Modify existing records.

UPDATE students SET age = 21 WHERE id = 1;

DELETE

  • Remove specific records.

    DELETE FROM students WHERE id = 1;

  • All Data from a Table

    To remove all rows without dropping the table:

    DELETE FROM table_name;

3. DQL (Data Query Language)

SELECT

Retrieve data from the database.

Basic SELECT

SELECT * FROM students;

SELECT Specific Columns

SELECT name, email FROM students;

Using WHERE Clause

SELECT * FROM students WHERE age > 18;

ORDER BY Clause

SELECT * FROM students ORDER BY age DESC;

GROUP BY Clause

SELECT age, COUNT(*) FROM students GROUP BY age;

LIMIT Clause

SELECT * FROM students LIMIT 5;

JOINs in SQLite

Combine rows from two or more tables.

SELECT students.name, courses.course_name
FROM students
INNER JOIN enrollments ON students.id = enrollments.student_id
INNER JOIN courses ON enrollments.course_id = courses.id;

4. TCL (Transaction Control Language)

SQLite supports basic transaction control to ensure data integrity.

BEGIN TRANSACTION

BEGIN TRANSACTION;

COMMIT

Save all changes made during the transaction.

COMMIT;

ROLLBACK

Undo changes made during the transaction.

ROLLBACK;

5. Other Useful SQLite Commands

Show All Tables in Database

Use the SQLite Master Table to list all tables.

SELECT name FROM sqlite_master WHERE type='table';

Get Table Structure

Use the PRAGMA table_info command to see column names, types, and constraints of a table.

PRAGMA table_info('students');

Create an Index

Indexes help in faster searching and ordering of data.

CREATE INDEX idx_name ON students(name);

List All Indexes of a Table

Use PRAGMA to list all indexes created on a table.

PRAGMA index_list('students');

Get Detailed Info About an Index

See which columns are covered by an index.

PRAGMA index_info('idx_name');

Drop an Index

Delete an existing index.

DROP INDEX idx_name;

Check Foreign Key Support

SQLite supports foreign keys but they must be enabled manually. 1 = enabled, 0 = disabled. 

PRAGMA foreign_keys;

Disable Foreign Key Constraints

We have to turn off the foreign key constraints temporarily because it gives us more flexibility in making changes.

PRAGMA  foreign_keys = OFF;

Enable Foreign Key Constraints

We can now turn foreign key constraints back on for ensuring data integrity.

PRAGMA foreign_keys = ON;

View All Constraints of a Table

There is no direct command, but you can get some information using:

SELECT sql FROM sqlite_master WHERE type='table' AND name='students';

VACUUM Database

Rebuild the database to reduce file size and defragment.

VACUUM;

Important Notes for SQLite

  • SQLite databases are stored in a single file with extensions like .db, .sqlite, .sqlite3, .db3.
  • Data types are dynamic; columns accept multiple types if not restricted.
  • There are no user access controls (no GRANT/REVOKE).
  • All commands can be separated by semicolons (;) when running multiple queries.
